# dev-0.0.9 Implementation and Release Record

Date: 2026-07-14 (Asia/Shanghai)

## Scope

- Replaced conversation-id-only execution with composite workspace/conversation targets and independent Windows utility or WSL process-group runtimes, preserving background work across workspace switches.
- Added receipt-backed, exactly-once Guide steering plus persisted public work runs that remain expanded while active, fold to elapsed time when complete, and retain later expand/collapse choices even after worker eviction.
- Limited expanded work records to filtered public streamed natural language, Guide lifecycle, and tool display name/status; hidden reasoning, tool arguments, call IDs, commands, paths, and results are excluded before IPC and persistence.
- Added cooperative checkpoint stop followed by target-only force restart, generation quarantine, late-descendant cleanup, target/generation-owned helpers, and strict all-settled backend shutdown.
- Unified editor/Markdown transitions, dirty Save/Discard/Cancel behavior, stale-open rejection, and complete close/reset semantics.
- Replaced PDFium access to `newmark-preview://` with a loopback-only, owner-bound, expiring PDF capability server supporting GET, HEAD, and single Range requests.
- Added Newmark-native Browser-Use over the same prewarmed embedded Browser guest with nine structured actions, opaque references, fixed isolated-world programs, physical-page serialization, host-side Plan revalidation, and private-field redaction.
- Added the Newmark-icon startup prewarm surface, seven-field renderer hydration barrier, update discovery during prewarm, deferred newer-version prompts, and restored the Archive icon.

## Known Distribution Boundary

- Windows MSI and executable are unsigned. Users and managed environments may see normal publisher/reputation warnings.
- The compatibility-locked Electron 33 runtime is covered by current high-severity Electron advisories; the full npm graph reports 11 high findings. Available audit remediation requires a major move to Electron 43 and electron-builder 26, which is deferred from this dev-0.0.9 line. Treat this package as a development prerelease and avoid untrusted content or privileged deployment until the runtime upgrade is completed.
- Windows utility runtimes do not yet use a Job Object. Zero residue is runtime-tested for the exercised Node/csc/PowerShell descendant cases with creation-identity checks, anchored rescans, and fail-closed quarantine; it is not claimed as an OS-enforced formal guarantee for every possible future process tree.
