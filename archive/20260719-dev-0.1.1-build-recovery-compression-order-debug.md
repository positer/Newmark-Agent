# dev-0.1.1 Build Recovery, Compression, and Task Order Debug

Date: 2026-07-19

## Fixes

1. Persisted Build recovery

Persisted `running` work runs are now migrated only at the cold persisted-state loading boundary. They become `interrupted`, retain their expanded transcript, and receive a frozen end boundary from the last persisted conversation update. This prevents a crashed or force-closed Build from continuing to accumulate duration when a later task is loaded. In-memory conversation switching does not rewrite a live cached run.

2. Compression frequency

Each compression stores the compacted character/token baseline. A subsequent request whose context still begins with the same compression summary must grow by a meaningful amount before another compression is allowed. The near-limit fallback pass is only forced after the first pass actually changed the context, preventing a skipped retry from becoming a second compression loop.

3. Historical task order

Compression model instructions, local fallback summaries, request-scoped task focus, and same-Build continuation prompts now explicitly process applicable unfinished historical tasks newest-to-oldest.
