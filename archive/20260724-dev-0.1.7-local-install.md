## User-visible repairs

- Editing a historical Build-start instruction stops only the targeted workspace/conversation/run, creates a new branch page, and immediately runs the edited instruction on that page.
- Branch selection replaces the cached Build-run snapshot, so the edited node and subsequent Build content do not leak beneath the active page.
- Clicking `<` or `>` is now a read-only branch inspection: it neither stops nor activates the runtime branch. Live events and completed send results are hidden while inspecting a sibling page, preventing cross-branch append leakage.
- Submitting from an inspected page first stops only the same conversation's running branch, activates the viewed page, and then starts the new Build. Other conversations are untouched.
- The pager is attached beneath the edited node and renders as inline clickable text such as `<2/2>`; the `<` and `>` text also supports keyboard activation.
- Persisted Guide rows expose one-click Copy and Edit actions.
- Editing a Guide creates a branch whose owning Build is copied only through the events before that Guide; the edited Guide and later events continue on the new page.
- Edited-file activity rows share the terminal activity row's 17 px icon column, text start, UI font, and compact content-width layout.
- Branch persistence honors `branchReset`, replacing truncated WorkRuns instead of merging the old tail back into the branch.

## Why the earlier attempted fix did not take effect

1. Pager placement selected the final user message rather than the edited message index.
2. Switching branches redrew chat messages but left the renderer's cached WorkRuns intact.
3. Persistence merged old WorkRuns into a branch even when the branch requested a reset.
4. Guide nodes lacked durable message-index hydration and therefore could not reliably expose Edit.
5. The first edited-file layout repair was still overridden by a later, more general `.conversation-work-file > summary` rule. The final repair raises the inline selector specificity and validates actual packaged pixel geometry.
