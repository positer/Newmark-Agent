# dev-0.0.10 native select contrast hotfix

Date: 2026-07-16 (Asia/Shanghai)

## Symptom and cause

The Windows native model popup was using the operating-system light popup surface while its option text inherited Newmark's dark-theme pale text token. Most unselected model rows and provider group labels therefore had very low contrast; the selected row remained readable only because Windows supplied a blue selection background.

The closed select control and the detached native popup are different rendering surfaces. A translucent glass background is appropriate for the closed control but is not a reliable popup palette. The fix therefore keeps the closed control unchanged and gives native popup rows explicit opaque colors plus a matching `color-scheme`.

## Implementation

- `DESKTOP/src/ui/index.html`
  - Adds separate dark and light semantic tokens for native select scheme, popup background, normal text, group text, selected background/text, and disabled text.
  - Applies the palette globally to `select`, `option`, and `optgroup`, covering the model toolbar, terminal, settings, Flow, automation, and dynamically generated selects.
  - Keeps native keyboard/accessibility behavior and does not replace selects with a custom widget.
- `DESKTOP/src/tests/visualPreferencesVerify.ts`
  - Parses both palettes from the built renderer.
  - Requires distinct dark/light backgrounds, matching native schemes, complete roles, and WCAG AA contrast for ordinary, group, selected, and disabled rows.
- `DESKTOP/scripts/release-ui-model-auto-context-smoke.cjs`
  - Exercises the grouped model selector in the packaged Electron application.
  - Reads real computed styles for the select, ordinary option, provider optgroup, and selected option under dark and light themes.
  - Composites transparent closed-control backgrounds before contrast calculation.
  - Terminates only the process tree it spawned, allowing safe use with `NEWMARK_TEST_EXE`.

## Packaged contrast results

| Theme | Closed control | Ordinary option | Provider group | Selected option | Native scheme |
| --- | ---: | ---: | ---: | ---: | --- |
| Dark | 12.55:1 | 16.02:1 | 10.07:1 | 5.71:1 | `dark` |
| Light | 16.88:1 | 15.28:1 | 6.77:1 | 6.57:1 | `light` |

All measured roles exceed the 4.5:1 acceptance threshold.
