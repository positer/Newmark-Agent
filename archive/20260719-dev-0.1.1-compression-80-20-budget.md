# dev-0.1.1 Compression 80/20 Budget

Date: 2026-07-19

## Change

Context compression now uses only the active model's context window. It triggers when estimated context reaches approximately 80% of the current deployment `max_tokens` and compacts toward 20%. The old fixed character threshold no longer triggers compression before the model-token boundary; its configuration key remains readable for compatibility.

The prior compacted-baseline growth guard remains active, so repeated tool receipts do not cause immediate recompression of the same retained context.

Model validation now also reads explicit maximum-context fields from the provider's model-list/catalog response. Valid positive limits are persisted to `ModelConfig.max_tokens`, which immediately becomes the source for Auto routing and the 80/20 compression budget. Output-token-only fields are not treated as context windows.
