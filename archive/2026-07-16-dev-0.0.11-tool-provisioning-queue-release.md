# Newmark Agent dev-0.0.11 release audit

Date: 2026-07-16 (Asia/Shanghai)
Scope: tool provisioning, chat and rewind reliability, Queue-to-Guide delivery, settings readability, Windows build/install, and GitHub prerelease.

## Root causes

- The Agent eagerly attached the full tool registry even when most schemas were irrelevant.
- First-turn chat could reach a provider before a usable deployment was resolved; failures could collapse to an empty `[Error]` and persist a failed assistant placeholder.
- Rewind crossed process boundaries using a position that could resolve to an assistant entry instead of the exact persisted user message.
- Queue Guide delivery did not distinguish explicit rejection from an uncertain transport result. Reusing a cached rejected ID could reject forever; changing an uncertain ID could duplicate delivery.
- Same-ID retries appended duplicate Guide rows and old rejected UI state could hide a successful retry.
- General settings still exposed native-looking action and number controls without separate dark/light semantic colors.
- Historical release smokes coupled compatibility to literal version 0.0.10 and did not click the new Guide action.

## Implemented boundaries

- `tool_provision` exposes a compact catalog of every callable tool while initially attaching at most eight task-relevant schemas.
- Grants use exact names, at most eight per call, sixteen dynamically provisioned schemas per run, and three broker calls. Search returns names only.
- Original schemas and names remain unchanged. ToolPolicy, platform, workspace, Native Tools, Utility/WSL/Electron host checks, schema validation, and side-effect rules remain authoritative.
- Broker traffic is not persisted as public chat, work-run activity, prompt text, tool arguments, or file content.
- Provider fallback re-resolves deployment identity and rebuilds the model-specific prompt/tool surface.
- Chat resolves a deployment before the request, retains a redacted non-empty provider diagnostic, and removes a failed trailing assistant from history.
- Rewind carries exact user-message identity through direct, Utility, and WSL boundaries.
- Editable renderer-owned queue rows expose a localized Guide button; backend queue mirrors remain display-only.
- Explicit rejection restores with a fresh request ID. Transport errors and missing receipts preserve the ID. A run-settled race restores without starting another turn.
- Same-ID uncertain retries reset only local optimistic state and reuse the existing rendered row; explicitly rejected attempts keep their audit row.
- Settings action, hover, focus, disabled, and number input states use semantic dark/light tokens and native `color-scheme`.
- The native editor uses synchronized dark/light background, gutter, text, syntax, caret, selection, inline-prediction, and completion tokens.
- Workspace and conversation selection reloads the current persisted snapshot instead of saving a stale pre-switch copy.
- File-change review rows use the same compact semantic action styling as the rest of the application.
- Packaged smoke requires broker constants, Queue recovery, a real Guide click and model continuation, a subsequent queue drain, and dark/light contrast of at least 4.5:1.
