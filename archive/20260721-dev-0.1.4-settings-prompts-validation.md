# dev-0.1.4 Release, Settings, Prompt Injection, Model Validation, and Linux Exit

Date: 2026-07-21 (Asia/Shanghai)

## Scope

Implement the requested `dev-0.1.4` changes without discarding the existing uncommitted queue, renderer, provider, or validation work:

1. Add adjacent General-settings actions to open and refresh the user-level `config.json`.
2. Add a live editor for the user-level `Agent.md` at the end of Models & Providers.
3. Improve global/workspace prompt injection ordering, normalization, cache invalidation, and deduplication.
4. Fix model-validation false negatives where an otherwise usable text model was marked unavailable because an optional Standard capability was unsupported.
5. Fix Linux explicit/tray exit leaving a ghost process that holds the single-instance lock.
6. Build, install, push, tag, and publish the cross-platform prerelease.

## Implementation

- `ConfigManager.reload()` reparses the disk configuration and clears stale workspace overrides before the active workspace overrides are reapplied.
- Desktop IPC opens `~/.Newmark/config.json` with the OS default application. Refresh is blocked during an active Agent turn; otherwise it reloads configuration, stops isolated runtimes, clears the conversation runtime pool, and reloads the renderer.
- Desktop and HTTP surfaces provide bounded read/write APIs for `~/.Newmark/agent.md`. The renderer uses a 450 ms debounced save and reports success/failure without exposing the file contents elsewhere.
- Global and workspace prompt saves invalidate the Agent system-prompt cache immediately.
- Prompt layers normalize BOM, CRLF, surrounding whitespace, and exact duplicate content. Global `Agent.md` is the user baseline and workspace `agent.md` is the more specific refinement; intrinsic Newmark safety/tool/permission policy remains authoritative.
- Model validation now bases aggregate availability on health plus the base text capability. Optional Standard/Extended failures remain visible in their capability records and produce `degraded`, not a global `invalid_config`/`unavailable`, when text remains usable.
- Fresh legacy false-negative cache entries self-repair to `degraded` without new provider requests.
- The model settings UI maps `verified` and `degraded` to the usable style; degraded is localized as available with limited capabilities.
- Package and lockfile versions are `0.1.4`.
- Tray Exit and renderer exit share one graceful shutdown path with a 12-second forced `app.exit(0)` deadline, preventing a stuck child resource from keeping the application and single-instance lock alive indefinitely.
- The Linux release builder writes latency evidence to a version-specific archive file rather than overwriting an older release record.
- The transitive `brace-expansion` dependency is updated to a non-vulnerable lockfile version.
